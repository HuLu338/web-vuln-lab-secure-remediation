## Vulnerability Matrix

| #   | Vulnerability                                        | Description                                                               | Vulnerable Behavior                                                    | Secure Fix                                                                  | Verification                                                                                  |
| --- | ---------------------------------------------------- | ------------------------------------------------------------------------- | ---------------------------------------------------------------------- | --------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------- |
| 1   | Broken Access Control                                | A user can access another user's file by manipulating the file ID.        | File access is handled only by object ID without ownership validation. | Ownership validation is enforced before file access, download, or deletion. | Unauthorized access to `/secure/files/{id}` returns `403 Forbidden`.                          |
| 2   | Insecure File Upload                                 | The system may accept unexpected or unsafe files if validation is weak.   | Uploaded files are accepted without strict filtering.                  | Extension allowlist and maximum file size restriction are enforced.         | Unsupported file types and oversized files are rejected.                                      |
| 3   | Stored XSS                                           | Stored malicious content may execute in another user's browser.           | User comments are rendered in an unsafe way.                           | Escaped output rendering is applied to comment content.                     | Script payload is displayed as plain text instead of executing.                               |
| 4   | Horizontal Unauthorized Access (IDOR-style behavior) | A user accesses another user's file via a modified request.               | `/vuln/files/{id}` returns another user's file information.            | `/secure/files/{id}` enforces ownership or admin authorization.             | Bob can access Alice's file in the vulnerable endpoint but is blocked in the secure endpoint. |
| 5   | Vertical Privilege Escalation                        | A normal user accesses admin-only audit functionality.                    | `/vuln/admin/audit` allows non-admin access.                           | `/secure/admin/audit` checks for admin role before access.                  | Non-admin access is blocked with `403`, while admin access is allowed.                        |
| 6   | Missing Security Event Visibility                    | Security-relevant actions are harder to analyze without a dedicated view. | Attack and remediation actions are not clearly grouped for review.     | A Security Events page displays access-control-related events.              | Security events such as denied and allowed actions can be reviewed in the UI.                 |
| 7   | Lack of Regression Protection                        | Vulnerability fixes may break later without automated verification.       | Security behavior is checked only manually.                            | Integration tests verify vulnerable and secure behavior automatically.      | `AccessControlIntegrationTest` validates access control and logging behavior.                 |
